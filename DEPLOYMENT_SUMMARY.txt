STEP 1: PUSH TO GITHUB
───────────────────────────────────────────────────────────────────────────────
git init
git add .
git commit -m "Prepare for Render deployment"
git remote add origin https://github.com/YOUR_USERNAME/kuraz-ai-backend.git
git push -u origin main

STEP 2: CREATE RENDER ACCOUNT
───────────────────────────────────────────────────────────────────────────────
→ Go to: https://render.com
→ Sign up (can use GitHub account)
→ Connect your GitHub account

STEP 3: CREATE POSTGRESQL DATABASE
───────────────────────────────────────────────────────────────────────────────
→ Click "New +" → "PostgreSQL"
→ Name: kuraz-ai-db
→ Region: Choose closest to your users
→ Instance Type: Free
→ Click "Create Database"
→ COPY the "Internal Database URL" (you'll need this!)

STEP 4: CREATE WEB SERVICE
───────────────────────────────────────────────────────────────────────────────
→ Click "New +" → "Web Service"
→ Select your GitHub repository
→ Configure:
   Name: kuraz-ai-backend
   Runtime: Python 3
   Build Command: pip install -r requirements.txt
   Start Command: uvicorn app.main:app --host 0.0.0.0 --port $PORT
   Instance Type: Free

STEP 5: SET ENVIRONMENT VARIABLES
───────────────────────────────────────────────────────────────────────────────
In your web service, go to "Environment" tab and add:

   DATABASE_URL          = [Paste PostgreSQL Internal URL from Step 3]
   GEMINI_API_KEY        = [Your Gemini API key]
   RESORT_NAME           = Tana Lakeside Resort
   RESORT_TOTAL_ROOMS    = 120
   PYTHON_VERSION        = 3.11.0

→ Click "Save Changes"

STEP 6: DEPLOY
───────────────────────────────────────────────────────────────────────────────
→ Click "Manual Deploy" → "Deploy latest commit"
→ Wait 3-5 minutes for deployment
→ Watch the "Logs" tab for progress

STEP 7: INITIALIZE DATABASE
───────────────────────────────────────────────────────────────────────────────
Once deployed, seed your database:

curl -X POST https://your-app-name.onrender.com/api/seed

STEP 8: TEST YOUR DEPLOYMENT
───────────────────────────────────────────────────────────────────────────────
# Health check
curl https://your-app-name.onrender.com/api/health

# View API docs
Open: https://your-app-name.onrender.com/docs

# Run test script (Linux/Mac)
./test_deployment.sh https://your-app-name.onrender.com

# Run test script (Windows)
test_deployment.bat https://your-app-name.onrender.com

🔧 TROUBLESHOOTING:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Build Fails:
   → Check Logs tab in Render dashboard
   → Verify requirements.txt is correct
   → Ensure Python version matches runtime.txt

Database Connection Error:
   → Verify DATABASE_URL is the Internal URL (not External)
   → Check database is in same region as web service
   → Ensure database is running

App Crashes:
   → Check all environment variables are set
   → View Logs tab for Python errors
   → Verify GEMINI_API_KEY is valid

Slow First Request:
   → Free tier sleeps after 15 minutes of inactivity
   → First request takes 30-60 seconds to wake up
   → Upgrade to paid tier or use UptimeRobot to keep alive
